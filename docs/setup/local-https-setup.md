# Local HTTPS Development with mkcert

This guide enables HTTPS for local development using **mkcert** — a tool that creates locally-trusted development certificates.

---

## Why Local HTTPS?

Modern web standards increasingly require HTTPS even in development:

- `SameSite=Strict` cookie policy (GitHub OIDC flows, secure sessions)
- HSTS (Strict-Transport-Security) headers
- Some APIs only accept HTTPS (webhooks, external integrations)
- **Production parity** — test the same TLS + HTTP/2 behavior as prod

---

## One-Time Setup

### 1. Install mkcert

#### macOS

```bash
brew install mkcert nss
```

#### Ubuntu/Debian

```bash
sudo apt-get install -y mkcert libnss3-tools
```

#### Fedora/RHEL

```bash
sudo dnf install -y mkcert nss-tools
```

#### Windows (with Chocolatey)

```bash
choco install mkcert
```

### 2. Generate Local CA & Certificates

```bash
# Install local CA in OS trust store (one-time, may prompt for password)
mkcert -install

# Generate certificate for localhost (valid for 10 years)
mkdir -p infra/certs
cd infra/certs
mkcert localhost 127.0.0.1 "*.local"

# You now have:
#   localhost+2-key.pem   (private key)
#   localhost+2.pem       (certificate)
```

#### Output

```text
Created a new local CA at "/home/user/.local/share/mkcert"
Installed in the system trust store ✓
Installed in Firefox ✓
Installed in Java keystore ✓

Created a new certificate valid for 825 days, signed by "mkcert development CA"
The certificate is at "localhost+2.pem" and the key is at "localhost+2-key.pem"
```

### 3. Run Local Environment

```bash
# Certs are already mounted in docker-compose.yml
docker compose up -d

# Services now available:
#   https://localhost:443    (all services via nginx HTTPS)
#   http://localhost:80      (redirects to HTTPS)
```

#### Verify HTTPS is working

```bash
curl -v https://localhost/api/v1/records
# Should show certificate chain and 200 response
```

## Architecture

```text
┌─ Your Browser ──────────┐
│   https://localhost:443 │
└──────────┬──────────────┘
           │ TLS/HTTPS
           ▼
    ┌──────────────────┐
    │  nginx (reverse  │
    │   proxy on :443) │
    └──────┬───────────┘
           │ HTTP :8000/:8001/:8003/...
           ├─► ingestor:8000
           ├─► inference:8001
           ├─► dashboard:8003
           ├─► analytics:8005
           └─► [other services]
```

#### TLS termination at nginx means

- Browser ↔ nginx: HTTPS (encrypted)
- nginx ↔ services: HTTP (same Docker network, no encryption needed)
- Services don't need TLS configuration
- No application code changes required

## Certificate Details

### Viewing Certificate Info

```bash
# See what's in your certificate
openssl x509 -in infra/certs/localhost+2.pem -text -noout

# Check certificate dates
openssl x509 -in infra/certs/localhost+2.pem -noout -dates
```

### Certificate Validity

- **Validity:** 825 days (auto-generated by mkcert)
- **Domains:** `localhost`, `127.0.0.1`, `*.local`
- **Trust:** Automatically trusted by your OS, all browsers, and Java

### Regenerating Certificates

If certs expire or you add a new domain:

```bash
cd infra/certs
rm -f localhost+*.pem
mkcert localhost 127.0.0.1 "*.local"

# Restart nginx
docker compose restart nginx
```

## Troubleshooting

### "Certificate not trusted" in browser

**Issue:** mkcert CA not installed in OS trust store.

#### Solution

```bash
# Reinstall local CA
mkcert -install

# Restart browser
# Clear browser cache (Cmd+Shift+Delete on Chrome)
```

### "Connection refused" on <https://localhost>

**Issue:** nginx not running or listening on 443.

#### Solution

```bash
# Check if nginx is running
docker compose ps | grep nginx

# Restart nginx
docker compose restart nginx

# Check logs
docker compose logs nginx

# Verify certs exist
ls -la infra/certs/localhost+*.pem
```

### nginx log shows `SSL_ERROR_RX_RECORD_OVERFLOW`

**Issue:** Attempting HTTP on HTTPS port or port mismatch.

**Solution:** Use correct URL:

- ✅ `https://localhost` (443)
- ❌ `http://localhost:443` (won't work)

### Certificate chain verification fails

**Issue:** mkcert CA not in system trust store on Linux.

#### Solution (Linux)

```bash
# Check where CA is installed
mkcert -CAROOT
# Usually: ~/.local/share/mkcert

# Manually add to system trust store if needed
sudo cp ~/.local/share/mkcert/rootCA.pem /usr/local/share/ca-certificates/mkcert.crt
sudo update-ca-certificates
```

## Development Workflow

### Run Full HTTPS Stack

```bash
# Start all services (nginx on :443 HTTPS, :80 HTTP redirect)
docker compose up -d

# Access services
open https://localhost              # Dashboard
open https://localhost/api/v1       # API docs
```

```bash

uv run pytest tests/ -v

# Or with coverage
uv run pytest tests/ --cov=ingestor --cov-report=html
```

### Backup/Restore Over Local Network

```bash
# Backup still works (services are local, no HTTPS needed)
bash infra/scripts/backup.sh

# Restore
bash infra/scripts/restore.sh postgres backups/postgres/pg_data_pipeline_*.sql.gz
```

### Chaos Testing

```bash
# Chaos scenarios run over Docker network (no TLS)

```

## CI/CD Note

#### GitHub Actions & deployment environments don't need mkcert

- CI runners generate certs via Let's Encrypt or cloud provider CA
- Production uses externally-signed certificates
- mkcert is **dev-only** (browser/OS auto-trusts, saves time)

## Production Deployment

When deploying to AWS/Kubernetes:

```dockerfile
# Production Dockerfile (not local dev)
# Use Let's Encrypt or AWS Certificate Manager certs
# No mkcert dependency
```

```yaml
# Kubernetes (example)
apiVersion: cert-manager.io/v1
kind: Certificate
metadata:
  name: app-cert
spec:
  secretName: app-tls
  issuerRef:
    name: letsencrypt-prod
```

## Common Issues & Solutions

| Issue | Solution |
|-------|----------|
| Browser shows "Not Secure" | Reinstall mkcert CA: `mkcert -install` |
| `curl https://localhost` fails | Check certs exist: `ls infra/certs/*.pem` |
| nginx can't find cert file | Verify docker-compose mounts: `docker inspect <nginx-id>` |
| Port 443 already in use | `lsof -i :443` to find process, then `kill -9 <pid>` |
| Certificate expired | Regenerate: `cd infra/certs && rm -f *.pem && mkcert localhost 127.0.0.1` |
